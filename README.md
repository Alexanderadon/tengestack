# TengeStack — зарплаты в IT Казахстана

Живой дашборд зарплатного среза IT-рынка КЗ по открытым вакансиям hh.kz.
Медианы по направлениям, грейдам и городам · востребованные технологии ·
любой срез шарится ссылкой · работает офлайн (PWA).

**Демо:** _(деплой — см. ниже)_ · **Методика:** `/methodology` · RU / EN

> ⚠️ Пока в репозитории лежит **синтетический** датасет для разработки
> (в UI висит баннер «демо-данные»). Реальные данные появляются после
> первой выгрузки: `pnpm data:refresh` с HH-токеном в `.env`.

---

## Зачем

Разработчики в Казахстане торгуются вслепую: зарплатные обзоры выходят раз в
год, устаревают к публикации и прячут сырые цифры за PDF с регистрацией.
TengeStack держит живой срез открытых вакансий с еженедельным обновлением —
и честно документирует, чему в этих данных верить нельзя.

## Архитектура

```
┌─────────────── еженедельно (CI cron) ────────────────┐
│ scripts/fetch.ts    HH API → data/raw/YYYY-MM-DD.jsonl│
│ scripts/build-dataset.ts                              │
│   merge архива → дедуп по id → нормализация →         │
│   колоночная упаковка → public/data/dataset.json      │
└──────────────────────┬────────────────────────────────┘
                       ▼
        статический хостинг (Vercel, без бэкенда)
                       ▼
┌──────────────────── браузер ──────────────────────────┐
│ загрузка снапшота с прогрессом (~100 КБ gzip)         │
│ движок фильтров: один проход по колонкам → Uint32Array│
│ KPI · SVG-чарты · виртуализированная таблица          │
│ фильтры ⇄ URL (history.replaceState, без роутера)     │
│ service worker: SWR датасета → офлайн                 │
└───────────────────────────────────────────────────────┘
```

**Сервера нет вообще.** Датасет — статический снапшот, вся аналитика
(фильтры, медианы, перцентили, гистограммы) считается в браузере.

## Ключевые решения и компромиссы

| Решение | Цена | Почему всё равно да |
|---|---|---|
| **Колоночный формат + словари** вместо массива объектов | кастомный кодек (~100 строк + тесты) | 6 200 строк = 0,66 МБ raw / **101 КБ gzip**; фильтры без создания объектов; CSR-скиллы как в разреженных матрицах |
| **Статический снапшот** вместо API | данные обновляются раз в неделю, не realtime | зарплатные агрегаты не нуждаются в realtime; хостинг бесплатный; офлайн бесплатен |
| **Фильтры на главном потоке**, без веб-воркера | теоретический потолок | замер: **0,03 мс** на проход по 6,2k строк → и 100k держит с запасом; воркер = complexity budget не туда |
| **Свои SVG-чарты** вместо recharts/d3 | нет готовых осей/легенд | −120 КБ бандла; полный контроль a11y (бины — фокусируемые кнопки); First Load JS всей страницы — **125 КБ** |
| **Скиллы из сниппетов** словарём, не из key_skills | занижение доли для длинных JD | key_skills = +1 запрос на каждую вакансию к API; сниппеты покрывают большинство упоминаний; ограничение задокументировано в методике |
| **Без налоговой нормализации** gross/net | ~10–15% шума в медианах | формула ИПН/ОПВ/ВОСМС дала бы ложную точность на данных-«вилках»; флаг gross сохранён колонкой |
| **Имена в URL**, не индексы словарей | URL длиннее | ссылки переживают еженедельную пересборку датасета |
| **Грейд «1–3 года» без слова в названии → unknown** | часть строк без грейда | честнее, чем записывать всех в джуны и портить медианы грейдов |

## Цифры

- Датасет: 6 200 вакансий → **0,66 МБ** raw, **101 КБ** gzip (колоночная упаковка ×3–5 к row-JSON)
- Полный проход движка фильтров: **~0,03 мс** (замер, Node 22; линейный по n)
- First Load JS: **125 КБ** (Next 15 static export, ноль чартовых библиотек)
- В DOM живёт ~20 строк таблицы из тысяч (`@tanstack/react-virtual`)
- 41 юнит-тест на нормализацию/таксономию/кодек/движок фильтров

## Запуск

```bash
pnpm install
pnpm data:fixture   # синтетика для разработки (или data:refresh с токеном)
pnpm data:build     # сборка колоночного снапшота
pnpm dev
```

### Реальные данные (HH API)

HH закрыл анонимный доступ к API (403). Нужен токен приложения:

1. Зарегистрируй приложение: https://dev.hh.ru/admin
2. Скопируй `.env.example` → `.env`, заполни `HH_CLIENT_ID` / `HH_CLIENT_SECRET`
3. `pnpm data:refresh` — выгрузка (~15–20 мин из-за вежливого rate-limit) + пересборка

Синтетика при этом отбрасывается автоматически.

## Стек

Next.js 15 (App Router, static export) · TypeScript strict · SCSS Modules ·
кастомные SVG-чарты · @tanstack/react-virtual · Node-пайплайн (tsx) ·
Vitest · GitHub Actions (CI + weekly data refresh) · PWA (ручной SW).

Архитектура фронта — FSD-light: `entities/` (домен) → `features/` (фильтры,
селекторы) → `widgets/` (чарты, таблица) → `app/`.
